Attribute Details
public string
  performedReason
Notes: Text specifying the motivation, cause, or rationale of an Activity. <br /><br />The datatype for this is string because the notion of  HL7's reasonCode may be moved to ActRelationship.<br /><br />For example, re-test, unscheduled or termination. For unscheduled, the event that occurred was initiated by a study participant and did not occur for protocol-specified reasons.<br /><br />For example, reasons that might qualify for being coded in this field might be: "routine requirement", "infectious disease reporting requirement", "on patient request", "required by law".
public string
  missedReason
Notes: The text that describes the rationale behind why a PerformedActivity is not done.<br /><br />This captures SDTM's REASND Domain. At present there is no coded set of values.  In HL7, there is a list of permissible missing value types, and we need to ensure that HL7's list is a superset of what is needed by SDTM.
public dateTime
  performedEndDateTime
Notes: The point in time when this activity ended.
public dateTime
  performedStartDateTime
Notes: The point in time when this activity began.<br /><br />For example, the datetime when sample was taken from the subject. <br /><br />For example, a dose of chemotherapy was given on June 12th starting at 9am and finishing at 12pm.

<p>Business Rules: <br/></p><p>An instance of a PlannedActivity associated with a PlannedStudySegment must be placed in/associated with a single PlannedSoACell. More than one instance of a single PlannedActivity is not allowed to occur within a single PlannedSoACell.  Stated another way:  the condition in which an instance of the activity exists in both a PlannedStudySegment and in a PlannedSubjectStudyContact such that the resulting PlannedSoACell would end up with two instances of the PlannedActivity is not allowed because a PlannedSoACell can only have one PlannedActivity.<br/></p><p><br/></p><p>For example, if a CBC is planned to occur within PlannedStudySegmentA and a CBC is also planned to occur within PlannedSubjectStudyContactX such that a single PlannedSoACell would, if it were to be constructed from the content of both PlannedStudySegmentA and PlannedSubjectStudyContactX, contain two PlannedCBCs, a condition that is not allowed (not to mention fundamentally inefficient and ineffectual.)<br/></p><p><br/></p><p>NOTE:  An instance of a PlannedActivity associated with a PlannedSoACell does not need to be associated a PlannedStudySegment.  For example, contingent activities need not to be in PlannedStudySegments.<br/></p>

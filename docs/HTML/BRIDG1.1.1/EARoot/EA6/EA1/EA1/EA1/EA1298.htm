<p>An identifier that uniquely identifies a thing or object. Examples are object identifier for HL7 RIM objects, medical record number, order id, service catalog item id, Vehicle Identification Number (VIN), etc. <br/></p><p><br/></p><p>For example: sponsor protocol number, site id, investigator id, subject id, accession id, etc. <br/></p><p><br/></p><p>NOTE: The assigningAuthorityName attribute is necessary to provide a context on who set that id for the study id, investigator id, etc.<br/></p><p><br/></p><p>NOTE: Similar to InstanceIdentifier data type of HL7 v3.  <br/></p>

Attribute Details
public string
  root
Notes: An unique identifier that guarantees the global uniqueness of the instance identifier. The root alone may be the entire instance identifier.
public string
  extension
Notes: A character string as a unique identifier within the scope of the identifier root.
public string
  assigningAuthorityName
Notes: A human readable name or mnemonic for the assigning authority. The Assigning Authority Name has no computational value. The purpose of a Assigning Authority Name is to assist an unaided human interpreter of an II value to interpret the authority. <br /><br />For example, the name of the organization who issued the extension.<br /><br />NOTE: No automated processing must depend on the assigning authority name to be present in any form.
public boolean
  displayble
Notes: Specifies if the identifier is intended for human display and data entry (displayable = true) as opposed to pure machine interoperation (displayable = false).

<p>Any action that can, in the context of a study, be planned, scheduled or performed.  <br/></p><p><br/></p><p>For example, a surgical procedure, a laboratory test, or the administration of the drug.<br/></p><p><br/></p><p>NOTE: An activity may be related to other activities in arbitrarily complex ways using instances of ActivityRelationship.<br/></p>

Attribute Details
public II
  identifier
Notes: The unique identifier assigned to an activity by some assigning body. <br /><br />For example, Internal or external ECG identifier, UUID for external ECG Waveform File.
public CD
  nameCode
Notes: A coded explanation of the activity. <br /><br />For example, a surgical procedure might be described with CPT4 or SNOMED term.<br /><br />For example, in a lab test, this coded value would be associated with a single analytic procedure (and the property of the results).  The textual description of the analytic test is captured in the complex data type CD.
public ST
  textDescription
Notes: A free text explanation of the activity.  <br /><br />NOTE: This may contain more detail than the description present in the text part of a coded concept.  
public CD
  categoryCode
Notes: Identifies the kind of activity, drawn from a controlled list of possible values, which groups similar or related activities together.<br /><br />For example, anti-cancer medication or general concomitant medications. In the case where category = anti-cancer medication, the subcategory = radiotherapy and the name = full brain radiation.<br /><br />For example, in Procedure, the category would support "abdominal surgeries".<br /><br />For example, in AdministrativeActivity, the category would support 3 types: Disposition (off study, epoch completion), Milestone (informed consent, enrollment, registry, randomization) and Other (unblinding) activities. <br /><br />For example, for lab procedures, hematology, urinalysis, chemistry.
public CD
  subcategoryCode		 source=WHO (May 2008) - Intervention(s)
Notes: A subdivision that has common differentiating characteristics within a larger category.<font color="#ff0000"> </font><br /><br />For example, chemotherapy, radiotherapy, hormonal therapy, alternative therapy. In the case where category = anti-cancer medication, the subcategory = radiotherapy and the name = full brain radiation. <br /><br />For example, if categoryCode = Intervention, subcategoryCode could = Drug (including placebo), Device (including sham), Biological/Vaccine, Procedure/Surgery, Radiation, Behavioral (e.g., Psychotherapy, Lifestyle Counseling), Genetic (including gene transfer, stem cell and recombinant DNA), Dietary Supplement, etc.
public SET
  reasonCode
Notes: A code or free text specifying the motivation, cause, or rationale of an activity. <br /><br />For example, routine requirement, drug reaction, infectious disease reporting requirement, on patient request, etc.<br /><br />NOTE: One can use Activity.reason when the Activity is occurring in isolation or the reason is relatively straightforward. Or if the reason is more complex (described in context of why the Activity was performed), one can use ActivityRelationship.type linking 2 or more Activities to another.
public CD
  statusCode
Notes: A value representing the current state of the activity. <br /><br />For example, for a lab test, this would be the condition or stage in the lifecycle of the test. <br /><br />For example, ordered, performed, etc.<br /><br />For example, for the patient study calendar, the status attribute captures the states of a scheduled activity. These states can be scheduled, cancelled, and performed.<br /><br />NOTE: A state is a named phase (or potential phase) of an instance of a concept in its lifecycle.  <br /><br />NOTE: We are assuming that the state machine for the Act class in the HL7 v3 RIM is sufficient to describe the lifecycle of all model activities.
public IVL
  statusDateRange
Notes: The date and time span for when a particular status becomes active until it is no longer considered active, enforceable, relevant, etc.
public ST
  comment
Notes: Additional description about the activity.

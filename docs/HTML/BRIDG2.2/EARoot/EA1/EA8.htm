<p>How do I navigate through this Enterprise Architect BRIDG File?<br/></p><p><br/></p><p>The Project Browser enables you to navigate through the Enterprise Architect project space. It displays Packages, Diagrams, Elements and element properties.    Two components have BRIDG content in them sinc Release 1.1. They are<br/></p><p><br/></p><p>1.  BRIDG Business Process Model<br/></p><p>Shows the draft dynamic content of BRIDG v 1.0.  <br/></p><p><br/></p><p>         *  BRIDG Activity Diagrams -- these are examplar diagrams from the NCI/caBIG CTMS I Project. These         actiivty diagrams have not been harmonized into BRIDG model and therefore are tagged with a                                <Proposed> status.<br/></p><p><br/></p><p>        * BRIDG State Diagrams - these are also examplar diagrams.  They are drawn for the HL7 Reference            Information Model (Activity State Diagram), and as a result of multiple discussions with various clinical trial            stakeholders (Subject State Diagram).<br/></p><p><br/></p><p>       * BRIDG Sequence Diagram - example - this is an examplar sequence diagram from the NCI/caBIG CTMS I project.<br/></p><p><br/></p><p>2.  BRIDG Domain Model<br/></p><p>Shows the static elements of the BRIDG model.  This package has the UML class diagram of BRIDG v 2.0.  There are two main pacakges under the BRIDG Domain Model:<br/></p><p>* BRIDG Release 2.0 -- Static Classes  -- this package has 5 Views that have been built to help with reviewing the model in parts and pieces.  For the complete UML class diagram, open View 5,  named -- Complete - Classes, Attributes, Inheritance.<br/></p><p>* HL7/ISO Data Types - These are the Release 2 Complex Data types from HL7.  Some of these datatypes have been bound to the attributes in the BRIDG class diagram.<br/></p><p><br/></p>
